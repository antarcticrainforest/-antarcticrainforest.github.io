107 ! Description:
108 ! Determines whether a diagnostic is available to a particular
109 ! (internal model,section) version. Also checks the option code IOPN
110 ! Called by INACTR, PRELIM.
111 
112 ! Method:
113 
114 ! The decimal value of the version mask, VMSK, was read from the
115 ! ppxref file by GETPPX, each (model, section, item). The version
116 ! number of the (model, section) - NMASK - is obtained from the
117 ! H_VERS array.
118 
119 ! The procedure for checking diagnostic availability is as follows:
120 ! (1) Check whether the relevant internal model is included in the
121 ! submodel configuration, by examining the INTERNAL_MODEL_LIST
122 ! array; if not, return.
123 ! (2) Check whether NMASK=0 - this implies that the diag is
124 ! unavailable to any version. If so, return.
125 ! (3) Check whether the diag is available to the specified version.
126 ! The 'version mask' binary code in the STASHmaster specifies
127 ! which versions the diag is available to. Eg., if it is available
128 ! to vns. 1 and 3, but not vn.2, the version mask is 101. VMSK
129 ! is the decimal equivalent of the version mask (5 in this example).
130 ! TSTMSK therefore calculates the quantity
131 
132 ! IMOD = MOD(VMSK,2**NMASK)/2**(NMASK-1)
133 
134 ! If IMOD=1, the diag is available; if IMOD=0, it isn't.
135 
136 ! The option code is checked in accordance with the ppxref option
137 ! code definitions.
138 
139 ! Note for code developers adding new tests: LMASK is initialised to
140 ! .TRUE. at top of deck. A series of tests is made and if any one
141 ! test fails LMASK is reset to .FALSE. Therefore do not reinitialise
142 ! LMASK to .TRUE. anywhere otherwise you may inadvertently overwrite
143 ! a preceding .FALSE. setting or you may mislead future code writers
144 ! who may do so.
145 ! For this reason, all unnecessary LMASK=.TRUE. were removed at 5.1
146
